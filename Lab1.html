Laboratorio 1, 25 enero, 2018




Información


  • ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

    • Ya que asi se evitan request de mas al servidor, a la vez evitas que informacion inutil llegue a este mismo y oueda causar acumulacion de basura, ademas de agilizar los procesos.

  • ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

    • Alterando el html form con "novalidate" o evitando que se haga el request, en otro caso eliminando el js.

  • Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

    • Ya que posteriromente se puede proteger tu html y js para evitar esas practicas, ademas la infrmacion no se enviara hasta el servidor a menos que se haga el request o este validado entonces no causari problema alguno.

  • Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

    • No debería de ser utilizado de manera frecuente ya que es considerada una mala practica sin embargo en caso de ser necesario es conveniente utilizarlo

  • Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

    • Ya que se puede perder el contenido de la pagina por causa del color de la imagen o si esta no es del tamaño completo de la pantalla se subdivide y puede causar un efecto no deseado

  • Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

    • Dependiendo de lo que busques hacer y que atributo uses, pero preferentemente yo recomiendo el uso de px para fuentes y padding mientras que % solo para medidas en cuanto a objetos responsivos

  • ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

    • Ya que evita que le navejador ande leyendo comentarios y espacios optimizando de esta manera la velocidad de carga de la pagina web

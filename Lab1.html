Información


  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    • Internet es la Red de Redes, en cambio la World Wide Web es el sistema que usamos para acceder a Internet.

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • El método GET solicita una representación de un recurso específico. Las peticiones que usan el método GET sólo deben recuperar datos; HEAD pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta; POST se utiliza para enviar una entidad a un recurso en específico, causando a menudo un cambio en el estado o efectos secundarios en el servidor; el modo PUT reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición;DELETE borra un recurso en específico y PATCH es utilizado para aplicar modificaciones parciales a un recurso.

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    • Se debe utilizar el método POST ya que lo enviado será invisible para el usuario.

  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    • GET

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    • No, este código es un identificador para peticiones correctas.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    • No necesariamente. Este código indica que no se encontró el recurso pero no necesariamente porque no exista, sino que el desarrollador pudo haberlo escondido a su conveniencia.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    • Sí, porque generalmente cuando sucede esto es porque hay algún fallo en la programación, o se ha llevado a cabo algún cambio en la plantilla o tema del sitio web y no se ha tomado la precaución de hacer una copia de seguridad.

  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    • Un elemento desaprobado significa que ya no se recomienda su uso, algunos de los atributos despeciados son acronym,applet,basefont,big,center,dir,font,frame,frameset,noframes,strike,tt>,u.

  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    • En HTML5 la sintaxis es compatible con XML, se incluye el elemento DOCTYPE, se incorporan nuevos elementos y atributos, tiene mejores estructuras y formularios, lenguaje más simple, plataforma de APIs, etc.

  • ¿Qué componentes de estructura y estilo tiene una tabla?

    • Como estructura están las etiquetas table que es para definir la tabla, tr que es para definir las filas de la misma y td que es para establecer el numero de columnas. También existen th o thead que es para el encabezado de la tabla, así como tfoot para el pie y tbody para el cuerpo, y estilos existen aquellos que te permiten unir filas o columnas como lo es rowspan y colspan respectivamente, y otras etiquetas tradicionales como width, height, border-collapse para la separación entre las celdas, border, border-style, margin, padding, background.

  • ¿Cuáles son los principales controles de una forma HTML5?

    • Los controles number para la entrada de un número, range para elegir un rango, date para elegir la fecha en calendario, color, search para buscar, datalist para listas de datos, email, tel y url para llenar campos.

  • ¿Qué tanto soporte HTML5 tiene el navegador que utilizas?

    • El navegador Google Chrome soporta 520 puntos de 555 lo cual es muy bueno y destaca sobre mucho otros navegadores.

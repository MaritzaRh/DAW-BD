Información


  • ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

    • Ya que asi se evitan request de mas al servidor, a la vez evitas que informacion inutil llegue a este mismo y oueda causar acumulacion de basura, ademas de agilizar los procesos.

  • ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

    • Alterando el html form con "novalidate" o evitando que se haga el request, en otro caso eliminando el javascript.

  • Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

    • Ya que posteriromente se puede proteger tu html y js para evitar esas practicas, ademas la infrmacion no se enviara hasta el servidor a menos que se haga el request o este validado entonces no causari problema alguno.
